생각나는 아이디어를 날짜별로 적어두는 잡기장.
나중에 여기서 "적용할 만한 것"을 골라 `improvement_suggestions` 테이블에 제안으로 승격시키거나
`📈 주간 리뷰`에서 수동으로 추가한다.

## 기입 규칙
- 한 줄 혹은 한 단락으로 짧게. 완성도보다 떠오른 날 기록이 우선.
- 날짜 헤더 `## 2026-04-18` 밑에 불릿으로 추가.
- 구현 결정이 나면 여기서 지우지 말고 `→ status: 적용됨 (sim-2026...)` 같은 한줄 주석만 덧붙인다.

## 2026-04-18

- 전체 공고에 백테스트를 돌려 결과를 DB에 캐시로 저장하고, 신규 공고가 들어올 때 **수요기관 기준**으로 매칭되는 과거 백테스트 내역을 진행 중 공고 상세에 바로 보여주기. (지금은 요청 시점에 즉석 계산이라 반복 비용이 큼)
- 특정 수요기관의 DB 표본이 적을 때, API로 **그 기관만 콕 집어 더 과거 기간까지 개별 조회**할 수 있는지 먼저 확인하고, 가능하면 기관 단위 타겟 백필을 돌려 표본을 보강. (현재 `backfill-recent-3y`는 카테고리 전체 기간 창 적재라 희소 기관 보강엔 비효율)

## 2026-04-19

- 공고 검색 전용 페이지(탭). **수요기관명** 또는 **공고번호**로 전체 공고 검색 → 목록에서 공고 클릭 시 상세 패널 표시.
  - 상세 패널에는 공고 메타(기관·구분·계약방법·예산·개찰일 등)와 공고 결과(낙찰자·낙찰가·낙찰률) 동시 표시.
  - 해당 수요기관이 `agency_parent_mapping`에 approved로 매핑돼 있으면 그 기관의 **백테스트 내역**을 함께 보여주고, 예측 vs 실제 낙찰가 추이를 **꺾은선 차트**로 시각화.

## 2026-04-20

- shrinkage blend 관리 화면/프로세스. 예측 노트에 `"하위 n건 + 상위 'X' 앵커 K건 (자체 비중 p%)"` 라벨이 자동 부착되는 로직(Phase 1)을 관리자가 눈으로 보고 튜닝할 수 있게 하자.
  - 관리자가 **K(prior_strength) 값을 슬라이더/입력으로 조정**하면 동일 공고 집합에 대한 예측 분포(자체 비중 p%, 낙찰률 예측, 실제 낙찰률) 변화를 실시간 비교.
  - 수요기관별로 **n 범위별(0~2, 3~5, 6~10, 11~20, 21+) 예측 오차·낙찰 확률 추이 차트**를 보여줘 "K를 얼마로 두면 특정 n 구간의 낙찰률이 좋아지는지" 의사결정을 돕기.
  - 궁극적으로는 **K를 기관/카테고리별 적응형**으로 자동 추정하는 단계(backtest-driven)까지 가되, 지금은 먼저 수동 관리 화면만 두고 감을 잡자.

- 진행 중 공고 × 모의 입찰 상태 뷰. 아직 개찰이 안 끝난 공고 목록에서 공고를 클릭하면 **우리 시스템이 자동 생성해둔 mock_bid 내역**이 하단에 펼쳐지도록.
  - 공고 수집 직후 `auto-bid-pending`이 돌기 때문에 모든 진행 중 공고에 mock_bids 행이 존재한다는 전제를 둠 (빠진 공고는 "자동 입찰 미생성" 배지로 구분 표시).
  - 표시 항목: simulation_id, 고객 번호별 bid_amount / bid_rate, 예측 중심값·lower·upper, portfolio_win_rate, uncertainty_score, parent_used, shrinkage 메타(K, 자체 비중).
  - 가능하면 **경쟁사(top_winners_for_scope)** 분포 vs 우리 투찰가의 위치를 같은 패널에서 시각화 → "우리가 어디에 걸려 있는지" 한눈에 파악.
  - 개찰이 끝나고 `bid_results`가 들어오면 자동으로 won/lost/disqualified 라벨이 상단에 찍히게 동일 화면에서 전/후를 이어 볼 수 있도록.
