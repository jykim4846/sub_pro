### 1-3. 기관 부모-자식 shrinkage (예측 정확도 개선)

사용자 요청: 하위기관 공고 수가 적을 때는 상위기관 트렌드를 쫓고, 많아지면 자체 트렌드를 쓰는 자연스러운 전환.

**분석기에 기존에 있던 shrinkage를 재활용** 하되, peer pool을 "카테고리 전체"에서 "부모+형제 그룹"으로 축소하는 방식.

- `g2b_bid_reco/agency_analysis.py`: `AgencyRangeAnalyzer`에 `parent_pool_agencies`, `parent_name` 파라미터 추가. 기본 `prior_strength=4.0` 유지(기존 테스트 보호), 호출부에서 `K=10`(사용자 지정)으로 오버라이드.
- `g2b_bid_reco/db.py`: `AGENCY_SHRINKAGE_K=10`, `get_agency_parent_pool(db, agency)` 헬퍼 + `load_cases_with_shrinkage()` 유틸.
- `dashboard.py`: 세 예측 호출부가 `_build_analyzer()` 헬퍼로 통일. 부모 매핑이 있으면 자동으로 `parent_pool_agencies` + `prior_strength=10` 주입. 예측 노트에 `"shrinkage blend: 하위 n건 + 상위 'X' 그룹 n건 (K=10, 자체 비중 p%)"` 자동 표기.
- `g2b_bid_reco/cli.py` `_auto_bid_scope_worker`: 스코프 시작 시 `agency_parent_mapping` 인덱스 1회 로드, 공고별로 부모 pool 주입. `parent_used` 필드가 실제 값으로 채워짐 → 다음 일일 배치부터 mock_bids에 반영.

테스트: `tests/test_agency_analysis.py` 에 부모 pool 필터 검증 2건, `tests/test_simulation.py` 에 `get_agency_parent_pool` / `load_cases_with_shrinkage` 커버리지 6건 추가. 전체 `pytest tests/` → **52 passed**.

### 3-3. launchd 배치 설치 (선택)

회사 PC에서도 자동 수집을 돌리려면:

1. `/bin/bash` 를 **전체 디스크 접근**에 추가(⇧⌘G → `/bin/bash`).
2. 플리스트 템플릿 경로 치환 후 설치:
   ```bash
   PROJECT_DIR="$HOME/Desktop/sub_pro/sub_pro"   # 실제 경로로 맞춤
   sed "s#/Users/jongyeonkim/Desktop/sub_pro/sub_pro#${PROJECT_DIR}#g" \
       ${PROJECT_DIR}/scripts/launchd-daily-collect.plist.template \
       > ~/Library/LaunchAgents/com.sub_pro.daily-collect.plist
   launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.sub_pro.daily-collect.plist
   launchctl kickstart gui/$(id -u)/com.sub_pro.daily-collect    # 스모크 테스트
   ```
   단, 이 리포지터리에는 아직 Desktop-경로 고정 plist만 커밋돼 있을 수 있으니 실제 파일은 `scripts/launchd-daily-collect.plist.template` 가 기준 (없으면 `~/Library/LaunchAgents/com.sub_pro.daily-collect.plist` 커밋본을 참고).
